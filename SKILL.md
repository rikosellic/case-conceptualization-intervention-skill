# 个案概念化

本 skill 用于根据心理咨询或心理治疗材料，生成专业中文个案概念化报告，并在报告基础上为治疗师提供后续干预指导。

## 资料读取规则

默认把 Markdown 核心读书笔记当作主要知识来源。原始书籍和模板文件只是二级查询资料，用于解决用户提出的细节问题，而不是普通报告任务的必读材料。

1. 先读 `references/index.md`，确认资料类别和对应核心读书笔记。
2. 再读相关 `*.md` 核心读书笔记，并以这些笔记完成常规概念化、报告组织和干预建议：
   - `references/个案概念化理论/skills/`：形成 CBT 或心理动力学等概念化假设。
   - `references/个案概念化模板/skills/`：选择报告或督导汇报结构。
   - `references/干预/skills/`：识别既往干预，并生成后续治疗建议。
   - `references/诊断/skills/`：仅用于诊断假设、鉴别方向、风险提示和转介建议。
3. 只有在以下情况才回看同类别 `books/` 下的 PDF/DOC/DOCX 原件：
   - 用户明确追问原书、原模板、原表格或原技术细节。
   - Markdown 核心读书笔记不足以回答某个具体细节问题。
   - 需要精确技术步骤、练习引导语、表格字段或模板原文。
   - 需要核对诊断边界或鉴别要点。

不要把三类资料混用。理论资料负责“如何理解个案”，模板资料负责“报告如何呈现”，干预资料负责“识别治疗师之前运用了哪些干预技术并指导治疗师接下来如何工作”。诊断资料只作为辅助，不替代概念化。

## 工作流程

### 1. 明确任务与材料状态

先判断：

- 用户需要完整报告、简版概念化、指定流派报告、督导讨论版，还是只需要干预指导。
- 用户提供的是初访记录、咨询记录、治疗逐字稿、评估摘要、督导材料，还是报告草稿。
- 用户是否指定理论取向、报告模板或诊断系统。
- 材料中是否出现风险线索，例如自杀、自伤、伤人、虐待、精神病性症状、躁狂、严重物质使用、进食障碍风险、解离或保护性议题。

如果材料不足以形成完整报告，仍可生成谨慎版本，但必须在“需进一步评估的信息”中明确标出缺失内容。

### 2. 提取临床相关信息

将原始材料整理为：

- 基本信息与材料来源
- 主诉与求助动机
- 当前症状、困扰和功能受损
- 诱发事件与近期压力源
- 问题的发展过程
- 重要成长史、家庭史、依恋经验和关系经验
- 既往治疗史、诊断史和用药史
- 治疗师已经使用过的干预技术、治疗意图和来访者反应
- 咨询过程中的互动模式、阻抗、联盟变化和情绪反应
- 资源、优势与保护因素
- 风险线索与伦理注意事项

必须区分事实与假设。除非材料中有直接依据，不要把推测出的动机、诊断、创伤机制、人格模式或家庭动力写成确定事实。

### 3. 形成概念化

根据 `references/index.md` 选择最相关的理论核心读书笔记。个案概念化需要解释：

- 来访者为什么在此时求助。
- 表层问题、功能损害和主要困扰是什么。
- 哪些易感因素、发展经验或关系经验可能塑造了当前问题。
- 哪些近期因素诱发或激活了当前困难。
- 哪些认知、情绪、行为、关系、身体反应、系统和环境机制维持了问题。
- 哪些资源、优势和保护因素可以被治疗利用。
- 哪些假设仍不确定，需要进一步评估。

必要时使用最低限度框架：

- 表层问题
- 形成因素
- 诱发因素
- 维持机制
- 核心假设
- 资源与保护因素
- 治疗含义

### 4. 选择报告模板

使用 `references/个案概念化模板/` 中的 Markdown 核心读书笔记决定报告结构。

默认匹配逻辑：

- 认知行为取向：材料重点涉及症状、自动思维、回避、应对行为和可测量目标。
- 心理动力取向：材料重点涉及重复关系模式、防御、情感调节、早期关系、移情或无意识冲突。
- 家庭/系统取向：材料重点涉及家庭结构、互动循环、角色期待、代际模式、伴侣或家庭冲突。
- 依恋/创伤知情取向：材料重点涉及创伤、安全感、解离、羞耻、依恋损伤、身体化威胁反应或关系安全。
- 整合式取向：多个机制都很突出，或用户要求完整专业报告。

如果没有明确匹配模板，使用整合式个案概念化报告结构，并说明报告采用通用整合式结构。

### 5. 生成报告

默认输出语言为中文。默认报告结构：

1. 个案基本信息与材料来源
2. 主诉与当前主要困扰
3. 治疗记录摘要（含既往干预技术识别）
4. 重要发展史与关系经验
5. 当前问题表现与功能影响
6. 个案概念化
7. 风险因素、资源与保护因素
8. 治疗目标
9. 后续治疗难点
10. 干预路径与治疗师建议
11. 需进一步评估的信息

可以根据所选模板调整标题，但除非用户要求简化，否则应覆盖上述内容域。

写作要求：

- 写成专业个案报告草稿，而不是日常解释或教学讲义。
- 使用清晰的临床语言，避免不必要的术语堆叠。
- 在“治疗记录摘要”中概括会谈进程，并单独识别治疗师已经使用的干预技术。
- 对既往干预技术的识别必须基于记录中的治疗师话语、任务布置、互动方式或会谈结构；材料不足时写“记录中未能明确识别”。
- 对不确定内容使用“可能”“提示”“可假设”“仍需进一步评估”等表达。
- 不编造缺失的人口学信息、诊断、咨询次数、家庭史、创伤史或治疗结果。
- 不过度断言因果关系，优先使用概念化语言而非绝对判断。

### 6. 生成干预指导

干预指导必须基于已经完成的个案概念化报告，而不是直接从孤立症状跳到技术建议。

使用 `references/干预/` 中相关 Markdown 核心读书笔记识别记录中已经出现的干预技术，并指导面向治疗师的后续建议。若采用整合式报告，选择能直接对应维持机制的干预方向。

干预指导应包含：

- 短期工作重点：安全、联盟、评估、稳定化、症状缓解和治疗框架
- 中期工作重点：核心维持机制、情绪/认知/关系模式、技能训练或经验加工
- 长期工作重点：人格或关系模式改变、复发预防、意义整合和生活结构调整
- 治疗师需要注意的难点：联盟风险、回避/阻抗、情绪过载、边界议题、反移情和风险升级
- 可使用的技术方向：只纳入被所选流派和个案概念化支持的技术
- 观察指标：如何判断干预是否有效，以及何时需要调整治疗策略

每条干预建议都应说明：

- 它针对概念化中的哪个机制。
- 它为什么适合来访者当前表现。
- 治疗师需要观察什么。

## 安全与临床边界

本 skill 用于生成专业报告草稿和督导取向的治疗建议，不替代正式诊断、急危机处理、医学建议、法律判断或现场临床督导。

当材料中出现高风险内容时：

- 单独加入风险部分。
- 建议治疗师立即进行结构化风险评估，并按照当地专业、伦理和法律要求寻求督导、转介或危机处理。
- 不提供可能促进自伤、伤人、胁迫或逃避照护的具体操作。

关于诊断：

- 只有当用户明确要求，或材料已经提供充分依据时，才使用诊断语言。
- 优先使用“诊断假设”或“鉴别方向”，避免在证据不足时给出确定诊断。

## 输出模式

如果用户没有指定模式，默认使用“完整版报告”。

- `完整版报告`：完整个案概念化报告加干预指导。
- `简版概念化`：简要概念化和关键治疗含义。
- `督导讨论版`：重点呈现假设、治疗过程、治疗师困境和下一次咨询策略。
- `指定流派版`：按照用户指定的理论取向和对应模板生成。
- `干预指导版`：仅当用户已经有概念化报告，并希望获得下一步治疗建议时使用。

## 最终回复检查清单

回复用户前，确认输出已经做到：

- 已先使用 `references/index.md` 和相关 Markdown 核心读书笔记。
- 只有在用户追问细节或摘要不足时才回看原始 PDF/DOC/DOCX。
- 区分事实、推论、假设和缺失信息。
- 在用户未指定流派时，说明选择的模板或取向。
- 在“治疗记录摘要”中包含治疗师已使用干预技术的识别；若材料不足，明确说明无法识别。
- 在概念化之后包含治疗难点和治疗师指导。
- 当材料涉及风险或信息不足时，明确提示风险和进一步评估需要。
